## db/connection.py — exports available to sub-modules
```python
# Every sub-module imports from connection using this pattern:
from .connection import _BACKEND, _conn, _cursor, _rows, _row, _ph, _ilike, _jp
```

Functions available:
  _BACKEND      str   — "postgres" or "sqlite"
  _conn()       contextmanager — yields connection; auto-commits or rolls back
  _cursor(conn) — returns cursor (RealDictCursor for postgres, Row for sqlite)
  _rows(cur)    — returns list of rows as dicts
  _row(cur)     — returns single row as dict or None
  _ph()         — returns "%s" (postgres) or "?" (sqlite)
  _ilike()      — returns "ILIKE" (postgres) or "LIKE" (sqlite)
  _jp(val)      — JSON parameter helper for JSONB

## Critical anti-patterns to avoid

WRONG — defines _P at sub-module level (evaluates before _init_backend() in sub-module context):
  _P = "%s" if _BACKEND == "postgres" else "?"  # BAD in any sub-module

CORRECT — use the helper function instead:
  ph = _ph()  # inside function body, evaluated at call time

WRONG — imports normalize from within db/ package:
  from . import normalize       # BAD — normalize.py is at project root, not in db/
  from db import normalize      # BAD

CORRECT — import from project root:
  import normalize              # normalize.py is at project root
