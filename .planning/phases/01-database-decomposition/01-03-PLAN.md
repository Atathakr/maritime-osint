## app.py current state (before this plan)
Line 30: app.secret_key = os.getenv("SECRET_KEY") or secrets.token_hex(32)
The `or secrets.token_hex(32)` clause silently generates a new session key on every restart —
sessions are invalidated on every Railway deploy, no error raised. This must be replaced.

## Replacement pattern (CONTEXT.md locked decision)
After load_dotenv(), before any Flask config or app = Flask(__name__):

```python
import sys

_secret_key = os.environ.get("SECRET_KEY")
if not _secret_key:
    print("[maritime-osint] SECRET_KEY is required. Set it in your environment or .env file. See .env.example.")
    sys.exit(1)
_app_password = os.environ.get("APP_PASSWORD")
if not _app_password:
    print("[maritime-osint] APP_PASSWORD is required. Set it in your environment or .env file. See .env.example.")
    sys.exit(1)

app = Flask(__name__)
app.secret_key = _secret_key
APP_PASSWORD = _app_password
```

Note: use os.environ.get() (not os.getenv()) to be explicit — both are equivalent.
The check runs AFTER load_dotenv() so .env files are respected.
